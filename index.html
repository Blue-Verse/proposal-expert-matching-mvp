핵심 요구사항 분석

  • 이중 회원 체계 — 수요자(서비스 신청)와 공급자(서비스 등록) 역할 분리, 이메일 인증 기반 가입/로그인, 마이페이지 통합 관리
  • 서비스 등록·중개 — 공급자의 카테고리별 서비스 등록, 수요자의 검색·필터링·상세 조회·신청 플로우 자동화
  • 결제·정산 자동화 — PG 연동(Toss/PortOne), 신청 즉시 결제, 공급자 정산 스케줄링, 환불·취소 정책 처리
  • 관리자(Admin) 대시보드 — 회원/카테고리/콘텐츠 관리, 결제 내역 조회, 정산 처리, 핵심 지표 통계 시각화
  • PC Web 반응형 — 데스크톱 최적화 + 태블릿/모바일 가독성 보장
  • 2차 글로벌 확장 대비 — i18n 다국어 구조 사전 설계, DB 스키마 다국어 컬럼 분리

제안 포인트

1. 자동화 중개 플랫폼의 핵심은 "프로세스 분기"

사람 개입을 최소화하기 위해서는 회원→서비스 등록→검색→신청→결제→정산까지의 상태 전이를 명확히 정의해야 합니다. 본 제안에서는 각 단계에 상태 머신(State Machine) 기반 도메인 모델을 적용하여, 비정상 케이스(결제 실패, 환불, 분쟁)도 자동 처리되도록 설계합니다.

2. 마켓플레이스 도메인 직접 경험 보유

강사↔회원 중개(피트니스 프랜차이즈), 1:N 매칭(디지털 명함 네트워킹), B2B 결재 워크플로우(전자결재 SaaS) 등 유사 플랫폼 3건 이상을 풀스택으로 수행한 경험을 그대로 적용 가능합니다. 이중 사용자 권한 모델(RBAC), Admin 대시보드, 결제·정산 처리 패턴이 모두 검증된 자산입니다.

3. MVP → 글로벌 확장의 단계적 로드맵

1차 MVP에서 i18n 다국어 구조와 다국어 DB 스키마를 사전에 설계하여, 2차 고도화 시 코드 재작성 없이 콘텐츠 추가만으로 글로벌 진출이 가능하도록 합니다. 이로써 1차에 추가 비용 없이 2차 확장 비용을 최소화합니다.

4. 1순위 "산출물 완성도"에 부합하는 검수 프로세스

각 마일스톤마다 화면 데모 + API 명세서 + 테스트 시나리오 검수를 진행하며, 최종 납품 시 소스 코드 + Figma 원본 + DB 스키마 + 배포 가이드 + 1개월 무상 하자보수를 제공합니다.

5. 안정적 유지보수·고도화 파트너십

월 단위 유지보수와 2차 다국어 고도화를 일관된 개발자가 책임지므로, 인수인계 비용 없이 장기 파트너십이 가능합니다.

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기능 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

다양한 분야의 서비스 공급자(전문가)와 해당 서비스를 필요로 하는 수요자(사용자)를 연결하는 자동화 중개 플랫폼을 구축합니다. 기존 유사 서비스와 차별화하여 사람의 개입을 최소화하고, 회원 가입부터 서비스 등록·검색·신청·결제·정산까지의 전 과정을 자동화함으로써 운영 효율성을 극대화합니다.

1차 MVP는 신속한 시장 검증을 목표로 하며, 출시 후 사용자 피드백을 반영하여 다국어 지원 등 글로벌 확장을 위한 2차 고도화로 이어갑니다.

2. 사용자 정의

사용자 유형설명주요 기능
수요자 (사용자)서비스를 검색·신청·결제하는 일반 사용자회원가입·로그인, 서비스 검색·필터, 상세 조회, 신청·결제, 마이페이지(신청 내역·결제 내역)
공급자 (전문가)서비스를 등록·관리하고 정산받는 전문가공급자 회원가입·심사, 서비스 등록·관리, 신청 접수·처리, 정산 내역 조회, 공급자 마이페이지
관리자 (Admin)플랫폼 전체를 운영·관리하는 운영자회원 관리, 카테고리·콘텐츠 관리, 결제 내역 조회, 정산 관리, 통계 대시보드

3. 기능 요구사항

3.1 회원 관리

  • [필수] 이메일 인증 기반 회원가입 (수요자/공급자 역할 분기)
  • [필수] 이메일/비밀번호 로그인, 비밀번호 재설정
  • [필수] 마이페이지 (프로필 수정, 비밀번호 변경, 회원 탈퇴)
  • [필수] 공급자 가입 시 추가 정보 입력 (사업자 정보·정산 계좌 등)
  • [선택] 소셜 로그인 (Google, Kakao) — 미팅 협의

3.2 서비스 등록·관리 (공급자)

  • [필수] 카테고리별 서비스 등록 (제목·설명·가격·이미지·옵션)
  • [필수] 등록 서비스 수정·중지·삭제
  • [필수] 신청 접수 알림, 처리 상태 변경
  • [필수] 정산 내역 조회

3.3 서비스 중개 (수요자)

  • [필수] 카테고리별 서비스 탐색
  • [필수] 키워드 검색 + 필터링 (가격대·평점·정렬)
  • [필수] 서비스 상세 조회 (공급자 정보·옵션·리뷰)
  • [필수] 서비스 신청 (옵션 선택·요청 사항 입력)
  • [선택] 찜·즐겨찾기, 리뷰·평점 — 미팅 협의

3.4 결제 시스템
